# 节点编排基础

GraphiteUI 的节点画布围绕几种核心节点展开：

- input 节点：负责接收文本或文件输入，并把内容作为输出传给下游。
- agent 节点：负责组合提示词、skill 和输出绑定，生成结构化结果。
- condition 节点：负责根据条件规则把流程分发到不同分支。
- output 节点：负责把最终结果显示出来，也可以承担持久化导出的职责。

常见交互包括：

- 双击画布创建节点。
- 从连接点拉线连接节点。
- 双击节点内可编辑文本来修改标题、简介或端口名称。
- 在 agent 节点里新增输入端口和输出端口。
- 拖拽文件到画布空白处，自动创建对应的 input 节点。

理解这些基础交互之后，再看 skill 和知识库能力会更容易，因为它们最终也只是被组织进这条节点链路里。
