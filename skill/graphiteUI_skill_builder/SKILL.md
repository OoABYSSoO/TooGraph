# GraphiteUI Skill Builder

Use this skill after the user requirement and expected behavior for a new custom GraphiteUI Skill are clear. It produces file contents only; it does not install, write, test, repair, package, or enable the generated Skill.

Lifecycle:
- `before_llm.py` injects the current `skill/SKILL_AUTHORING_GUIDE.md` so the LLM follows the active Skill protocol.
- `after_llm.py` normalizes the generated file-content fields and strips accidental Markdown fences.

Inputs generated by the LLM:
- `skill_json`: `skill.json` as a JSON object using `graphite.skill/v1`.
- `skill_md`: complete `SKILL.md` Markdown content.
- `before_llm_py`: complete `before_llm.py` source, or an empty string when not needed.
- `after_llm_py`: complete `after_llm.py` source, or an empty string when not needed.

Outputs:
- The same four fields, normalized for downstream display or a later controlled file-writing step.

Rules:
- The generated Skill must keep GraphiteUI state binding in the runtime, not inside skill scripts.
- The generated Skill must declare permissions in `skill.json` when it needs network, file access, browser automation, secret access, or subprocess execution.
- Do not generate zip files, paths, smoke-test reports, write commands, graph templates, or installation side effects.
- Do not include legacy fields such as `health`, `configured`, `healthy`, `targets`, `runPolicies`, `kind`, `mode`, `scope`, or `label`.
