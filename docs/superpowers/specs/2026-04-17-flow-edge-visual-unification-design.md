# 流程线统一视觉设计

## 目标

把编辑器里所有“表示流程”的线统一成同一套视觉和路径规则：

- 统一使用正交折线
- 统一使用圆角拐点
- 统一使用虚线
- 条件节点遵循“顶部输入、底部输出”

这次改动只处理前端显示与交互一致性，不改变正式协议。

## 背景

当前画布存在两套流程线表现：

- 条件 branch 流程线已经使用自定义正交圆角路径
- 普通控制流边仍然主要走默认 edge renderer

结果是：

- 普通流程边和条件流程边看起来不像同一种语义
- 条件节点虽然协议上已经是“顶部进、底部出”，但视觉表达仍然不够统一

## 设计结论

### 1. 流程线统一为一类 renderer

所有流程线都使用同一类自定义 edge renderer：

- 普通控制流边
- 条件 branch 控制流边

统一规则：

- 线型：虚线
- 路径：正交折线
- 拐角：圆角

不再让普通控制流边继续使用 React Flow 默认曲线。

### 2. 数据流线继续单独存在

数据流线不进入这次统一。

它们继续保持：

- 自动投影
- 不可编辑
- 明显区别于流程线

这样可以继续维持“控制流 / 数据流”双平面。

### 3. 节点口位统一

节点流程口统一为：

- `input`
  - 无顶部流程输入
  - 底部 1 个流程输出
- `agent`
  - 顶部 1 个流程输入
  - 底部 1 个流程输出
- `condition`
  - 顶部 1 个流程输入
  - 底部多个 branch 流程输出
- `output`
  - 顶部 1 个流程输入
  - 无底部流程输出

### 4. 条件节点约束

条件节点遵循：

- 左侧：数据输入 `reads`
- 顶部：流程输入
- 底部：branch 流程输出
- 右侧：不承载流程语义

branch 名称展示与双击编辑继续放在底部 branch 项上。

### 5. 路径细节

所有流程线共享同一套路径构造器，但允许来源偏移策略不同：

- 普通流程边：
  - 使用简单的默认起始偏移
- 条件 branch 流程边：
  - 保留“越靠下的 branch，第一段水平偏移越长”的避让规则

共同约束：

- 目标统一落到目标节点顶部流程输入
- 允许中间垂直段为上、下或 0
- 去除无意义的零长度拐点

## 实现边界

这次改动不做以下事情：

- 不改后端 schema
- 不改控制流 / 数据流正式协议
- 不改数据流投影算法
- 不改 branch 语义

## 验收标准

满足以下条件即视为完成：

1. 所有流程线都表现为虚线、正交折线、圆角
2. 普通流程边和条件 branch 流程边在视觉上属于同一体系
3. 条件节点只从顶部接收流程输入，只从底部输出 branch 流程
4. 数据流线继续与流程线明显区分
5. 现有流程编辑能力不回归
