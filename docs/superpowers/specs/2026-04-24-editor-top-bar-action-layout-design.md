# 编辑器顶部页签与运行胶囊重构设计说明

日期：2026-04-24

## 1. 背景

当前编辑器顶部由 [EditorTabBar.vue](/home/abyss/GraphiteUI/frontend/src/editor/workspace/EditorTabBar.vue) 承担两类职责：

1. 页签工作区  
   多图页签、激活、关闭、重命名、拖拽排序。
2. 图级操作区  
   `State / 新建图 / 从模板创建 / 打开已有图 / Save / Validate / 导入 Py / 导出 Py / Run`。

这导致顶部存在三个明显问题：

1. 顶部信息混层  
   “打开别的图”和“操作当前图”被塞进同一条控制条里，缺少明确主次。
2. 运行按钮下方仍有一整条顶栏底色  
   即使 `Run` 是最重要动作，它仍然附着在整条 toolbar 上，视觉上不像直接悬浮在画布上方。
3. 窄屏表现不符合当前心智  
   现有窄屏方案会把 controls 改造成另一套压缩布局；用户希望保留完整操作，只是把运行胶囊整体移到第二行并靠右显示。

用户已经明确确认的新方向：

- 左侧页签带保留
- 页签带增加浏览器式 `+`
- `+` 弹层统一承接：`新建空白图 / 从模板新建 / 打开已有图`
- 右上只保留“当前图动作”
- `Run` 是唯一主按钮
- `State` 是次主操作
- `Save / Validate / Import / Export` 改成图标按钮
- 右上动作区应是独立悬浮胶囊，下面直接露出画布
- 窄屏下不简化功能，完整胶囊落到第二行并靠右显示

## 2. 目标

### 2.1 核心目标

1. 把顶部重新拆成“页签工作区”和“当前图动作区”两套清晰结构
2. 让右上 `Run` 行从整条 toolbar 中脱离，变成独立悬浮胶囊
3. 把“新建/打开别的图”的入口收回页签体系，贴合浏览器式多页签心智
4. 在桌面和窄屏下都保持明确主次，不通过删功能来换布局

### 2.2 非目标

1. 本次不改图运行语义
2. 本次不改 `State / Run / Save / Validate / Import / Export` 的事件语义
3. 本次不新增“最近打开”“搜索图”“搜索模板”等扩展能力
4. 本次不重做整个页签视觉语言，只重构顶部职责分层和动作布局
5. 本次不为右侧侧栏额外新增复杂避让系统；若实现时发现真实重叠，再单独跟进

## 3. 用户确认后的产品结论

### 3.1 顶部结构结论

顶部应拆成两类独立区域：

1. 左侧页签带  
   负责当前工作区标签、拖拽、关闭、重命名，以及 `+` 入口。
2. 右上运行胶囊  
   负责当前图的所有动作，固定在编辑器右上角。

因此顶部不再保留“整条 controls 底板”的表达。

### 3.2 主次关系结论

用户明确确认：

- 所有现有动作都保留
- 但要有主次

最终主次关系如下：

1. `Run`：唯一主按钮
2. `State`：次主按钮
3. `Save / Validate / Import / Export`：普通工具动作

### 3.3 新建/打开图入口结论

用户明确要求把“新建/打开图”的入口放回页签体系，而不是留在右上动作区。

最终结论：

- 页签带末尾新增浏览器式 `+`
- `+` 点击后弹出小窗口
- 小窗口统一承接：
  - `新建空白图`
  - `从模板新建`
  - `打开已有图`

### 3.4 响应式结论

用户确认的窄屏规则：

- 不做按钮简化
- 不隐藏任何现有动作
- 顶部改成两行
- 第一行：页签带与 `+`
- 第二行：完整运行胶囊
- 第二行胶囊整体靠右显示

## 4. 新的界面结构

### 4.1 桌面端结构

桌面端顶部从视觉上分成两层：

1. 页签带  
   一条较薄的顶部条，仅服务页签本身。
2. 右上悬浮动作胶囊  
   固定在编辑器右上，独立于页签带存在。

画布从页签带下方直接开始。右上胶囊虽然压在画布上方，但不应在其下方再延续一整块 toolbar 背景。

### 4.2 右上悬浮动作胶囊

右上胶囊内的顺序固定为：

`工具图标组 ｜ State ｜ Run`

其中：

- 工具图标组：`Save / Validate / Import / Export`
- `State`：次主 pill，保留文案与数量
- `Run`：唯一高对比主按钮，固定在最右侧

设计原则：

- `Run` 作为最右侧锚点，形成稳定的主操作落点
- `State` 紧邻 `Run`，作为编辑器最常用次级动作
- 低频工具动作收进紧凑图标组，减少横向文字噪音

### 4.3 页签带与 `+`

页签带保留当前浏览器式页签交互：

- 激活
- 关闭
- 中键关闭
- 拖拽排序
- 双击重命名

在页签带末尾新增一个独立 `+` 按钮，视觉上像浏览器新标签页入口，而不是某个 tab 的一部分。

### 4.4 `+` 弹层

`+` 弹层采用轻量弹出层，不做大面板。

弹层内部按“入口优先”组织：

1. `新建空白图`
2. `从模板新建`
3. `打开已有图`

为了保持轻量和清晰，推荐结构为：

- 顶部是三个明确入口
- `从模板新建` 和 `打开已有图` 在点击后展开各自的紧凑选择区
- 选择区内部继续复用现有 `WorkspaceSelect` 或等价的 Element Plus 选择原语，而不是重写一套自定义搜索控件

这样既符合“先点 `+`，再选模式”的心智，也能复用现有模板/图列表数据流。

## 5. 组件职责重组

### 5.1 `EditorTabBar`

`EditorTabBar` 不再承担完整工具栏职责，而应收敛为“页签带容器”。

保留职责：

- 标签页渲染
- 标签页交互
- `+` 按钮与其弹层

移出职责：

- 当前图动作胶囊
- `Run / State / Save / Validate / Import / Export` 的视觉布局

### 5.2 新增独立动作胶囊组件

建议新增一个专职组件：

```ts
EditorActionCapsule.vue
```

负责：

- `State` 次主按钮
- `Save / Validate / Import / Export` 图标按钮
- `Run` 主按钮
- tooltip、顺序、悬浮容器样式
- 桌面/窄屏位置切换

这样能把“页签工作区”和“当前图动作区”彻底解耦，避免 `EditorTabBar.vue` 再次膨胀成全能顶部组件。

### 5.3 `EditorWorkspaceShell`

`EditorWorkspaceShell` 负责组合顶部的两类区域：

- 上方页签带
- 右上固定动作胶囊

同时负责：

- 为动作胶囊提供固定定位的挂载位置
- 在窄屏断点下把胶囊移到第二行
- 保证胶囊下方直接露出画布，而不是继续渲染一整条 controls 底板

## 6. 交互细节

### 6.1 `Run`

`Run` 仍保持当前点击语义不变，只调整视觉地位：

- 唯一主色
- 胶囊最右
- 始终可一眼定位

### 6.2 `State`

`State` 继续显示数量，但退成次主 pill：

- 保留 `State` 文案
- 保留数量徽标
- 不使用主按钮配色

### 6.3 工具图标

`Save / Validate / Import / Export` 改为图标按钮后，必须补足：

- tooltip
- 可访问名称
- 一致的 hover / focus 状态

图标语义建议：

- `Save`：磁盘/保存类图标
- `Validate`：检查/校验类图标
- `Import`：导入类图标
- `Export`：导出类图标

这些图标应优先复用现有 Element Plus 图标库，而不是手写 SVG。

### 6.4 `+` 弹层关闭规则

弹层应遵循常见 popover 规则：

- 点击外部关闭
- 选择任一动作后关闭
- 打开 `从模板新建` / `打开已有图` 子区后，仍保持轻量，不升级成全屏抽屉

## 7. 响应式行为

### 7.1 桌面端

- 页签带贴顶
- `+` 紧跟在页签带末尾
- 运行胶囊固定在右上角
- 胶囊下方直接露出画布

### 7.2 窄屏端

顶部改成两行：

1. 第一行：页签带与 `+`
2. 第二行：完整运行胶囊，整体靠右显示

注意：

- 不做按钮删减
- 不把图标组折叠进更多菜单
- 不把 `State` 或 `Run` 隐藏

### 7.3 极窄宽度

在极窄宽度下，完整胶囊仍保持原有动作集合。

为了满足“完整保留”而非“功能简化”，允许：

- 第二行容器保留右对齐
- 胶囊自身出现最小必要的横向滚动能力

不允许：

- 自动隐藏某些按钮
- 把工具图标二次折叠进 overflow 菜单

## 8. 视觉方向

### 8.1 页签带

页签带继续使用当前温暖纸面风格，但要进一步收敛职责：

- 背景只服务页签带本身
- 不再把右上工具区包进同一层底板
- 不再让顶部像一整条后台 toolbar

### 8.2 运行胶囊

运行胶囊的视觉目标是：

- 一枚明确压在画布上的轻量浮层
- 不厚重
- 不像第二条导航

建议表现：

- 较浅的半透明暖白背景
- 轻边框
- 很薄的阴影
- 圆角胶囊轮廓
- 内部按钮间距统一，但 `Run` 比其他动作更有重量

### 8.3 画布关系

用户明确要求“运行按钮的那一行下面应该直接是画布”。

因此视觉上必须满足：

- 右上胶囊之下没有延续的整条顶栏底色
- 胶囊不是嵌在一整块 header 面板里
- 胶囊与画布之间是“浮层压在上方”的关系

## 9. 测试与验收

### 9.1 结构测试

至少需要补或改这些结构测试：

1. `EditorTabBar` 只保留页签带与 `+` 入口，不再包含旧的完整 controls dock
2. `EditorTabBar` 的 `+` 弹层包含：
   - `新建空白图`
   - `从模板新建`
   - `打开已有图`
3. 新动作胶囊组件包含：
   - 图标工具组
   - `State` 次主按钮
   - `Run` 主按钮
4. `EditorWorkspaceShell` 在桌面端把胶囊作为右上独立浮层渲染
5. `EditorWorkspaceShell` 在窄屏端把胶囊放到第二行并靠右显示

### 9.2 交互验收

验收时需确认：

1. 点击 `+` 后能完成：
   - 新建空白图
   - 从模板新建
   - 打开已有图
2. `Run / State / Save / Validate / Import / Export` 原有事件语义不变
3. 桌面端右上胶囊下方直接露出画布
4. 窄屏端完整胶囊仍存在，且落在第二行靠右

## 10. 实施边界

本次实现建议控制在以下范围：

- 重构顶部组件职责
- 新增 `+` 弹层入口
- 新增独立动作胶囊组件
- 调整桌面/窄屏布局
- 同步更新结构测试

暂不进入：

- 最近打开历史
- 搜索型 launcher
- 复杂命令面板
- 胶囊与右侧面板的高级动态避让系统
